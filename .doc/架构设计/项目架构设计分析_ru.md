## 一、Обзор проекта

**EasyAIoT**（Cloud-Edge-Device Integrated Collaborative Algorithm Application Platform） — это **интеллектуальная IoT-платформа с интеграцией облака, периферии и устройств**, ориентированная на глубокое слияние AI и IoT. Видение проекта — «сделать AI доступным для всего мира без барьеров».

| Показатель | Данные |
|------|------|
| Всего коммитов | 1,760 |
| Основной контрибьютор | 翱翔的雄库鲁（3,988 commits，95%+） |
| Итерации версий | V1.0.0 → V9.17.0（всего 35+ веток версий） |
| Масштаб кода | Java 2,374 файла / Python 173 файла / Vue 558 файлов / TypeScript 610 файлов / C++ 30 файлов |
| Shell-скрипты | 79（автоматизация развёртывания/эксплуатации） |
| SQL-скрипты | 7（инициализация нескольких БД） |

## 二、Общая архитектура

### 2.1 Многоуровневая архитектура

```
┌─────────────────────────────────────────────────────────────┐
│                    WEB-фронтенд（Vue 3 + Ant Design Vue）        │
├─────────────────────────────────────────────────────────────┤
│                 API Gateway（Spring Cloud Gateway）           │
├──────────┬──────────┬──────────┬──────────┬─────────────────┤
│ iot-system │ iot-infra │ iot-device │ iot-dataset │ iot-message │
│  Управление системой  │  Инфраструктура  │  Управление устройствами   │  Управление наборами данных  │  Push-уведомления    │
├──────────┴──────────┴──────────┴──────────┴─────────────────┤
│          iot-sink（слой адаптации протоколов：MQTT/TCP/HTTP/EMQX）           │
├──────────┬──────────┬──────────┬─────────────────────────────┤
│  AI-сервис  │ VIDEO-сервис│ TASK-модуль │ iot-gb28181（протокол видеонаблюдения）  │
│ Flask+YOLO│ Flask+потоковая обработка│ C++-инференс  │   Java SIP-сигнализация            │
├──────────┴──────────┴──────────┴─────────────────────────────┤
│              Слой middleware（Nacos / PostgreSQL / Redis / Kafka / MinIO / TDengine）│
└─────────────────────────────────────────────────────────────┘
```

### 2.2 Разделение микросервисов

Проект использует **многоязычную микросервисную архитектуру**, разделённую на 5 основных модулей по доменам ответственности：

| Модуль | Язык/фреймворк | Ответственность | Кол-во сервисов |
|------|-----------|------|--------|
| **DEVICE** | Java 21 + Spring Boot 2.7 + Spring Cloud | Управление устройствами、управление системой、push-уведомления、наборы данных、хранение файлов | 8+ микросервисов |
| **AI** | Python + Flask + PyTorch + YOLO | Обучение моделей、инференс、развёртывание、OCR、речь、LLM | 1 основной сервис + подсервисы |
| **VIDEO** | Python + Flask + OpenCV + FFmpeg | Обработка видеопотоков、алгоритмы в реальном времени/снимков、запись、тревоги | 1 основной сервис + 6 подсервисов |
| **TASK** | C++17 + OpenCV + ONNX Runtime + FFmpeg | Движок инференса на периферии в реальном времени | Независимый процесс |
| **WEB** | Vue 3 + TypeScript + Vite + Ant Design Vue | Полнофункциональная платформа управления фронтендом | SPA |

## 三、Подробная техническая архитектура модулей

### 3.1 Модуль DEVICE（кластер Java-микросервисов）

**Технологический стек：**
- **Фреймворк**：Spring Boot 2.7.18 + Spring Cloud 2021.0.5 + Spring Cloud Alibaba 2021.0.4.0
- **JDK**：Java 21
- **Шлюз**：Spring Cloud Gateway
- **Регистрация/конфигурация**：Nacos
- **БД**：PostgreSQL（основная）+ TDengine（временные ряды）
- **ORM**：MyBatis-Plus 3.5.5 + Dynamic Datasource
- **Кэш**：Redis + Redisson 3.18.0
- **Очередь сообщений**：RocketMQ / Kafka
- **Объектное хранилище**：MinIO
- **Workflow**：Flowable 6.8.0
- **Планировщик задач**：XXL-Job 2.3.1
- **API-документация**：Knife4j 4.3.0 + SpringDoc
- **Мониторинг**：SkyWalking 8.12.0 + Spring Boot Admin
- **Утилиты**：Hutool 5.8.25、MapStruct 1.5.5、EasyExcel 3.3.3

### 3.4 Модуль TASK（C++-движок периферийного инференса）

**Технологический стек：**
- **Язык**：C++17
- **Сборка**：CMake + vcpkg
- **Движок инференса**：ONNX Runtime（GPU-ускорение）
- **Детекция объектов**：YOLOv11
- **Обработка видео**：OpenCV + FFmpeg（libavcodec/libavformat/libavutil/libswscale）
- **Логирование**：glog
- **JSON**：jsoncpp
- **Сеть**：libcurl（HTTP-callback）
- **Платформы**：Windows + Linux

**Архитектура：**
```
main.cpp → Manage (Server) → Config → ConfigParser
         → Yolov11Engine (推理引擎)
         → Yolov11ThreadPool (线程池)
         → Detech (检测逻辑)
         → Draw (绘制标注)
         → RTMPEncoder (RTMP 编码推流)
         → AlarmCallback (告警回调)
```

**Ключевые особенности：**
- Независимый процесс, управляется INI-конфигурацией
- Поддержка RTSP pull в реальном времени + YOLO-инференс
- Многопоточный пул инференса
- RTMP-кодирование и push-стриминг
- HTTP-callback для тревог
- Кроссплатформенность（Windows/Linux）

| SRS | latest | Медиасервер（пересылка RTSP/RTMP） |

### 4.2 Архитектура развёртывания

- **Единая оркестрация Docker Compose**：отдельный `docker-compose.yml` для каждого модуля
- **Единый скрипт установки**：`.scripts/docker/install_linux.sh` — развёртывание всех сервисов одной командой
- **Двухэтапная сборка**：Dockerfile.base（кэш Maven-зависимостей）→ Dockerfile каждого модуля
- **Поддержка GPU**：автоопределение GPU и включение NVIDIA Container Runtime
- **Поддержка ARM**：специальные скрипты установки и Dockerfile для ARM64
- **银河麒麟**：скрипты адаптации под отечественную ОС

### 4.3 Проектирование БД

- **6 PostgreSQL БД**：изоляция по бизнес-доменам
- **SQL-скрипты инициализации**：7 SQL-файлов в `.scripts/postgresql/`
- **Автоинициализация**：автоматическое выполнение через `initdb.d` при запуске Docker
- **Супертаблицы TDengine**：`.scripts/tdengine/tdengine_super_tables.sql`
